Power Estimation Attempt
------------------------
Full dynamic power estimation requires a VCD switching activity file generated
from gate-level simulation. A VCD file was not available for this run because
the co-simulation was performed at RTL level (tb_top.sv driving top.sv) rather
than at gate-level with back-annotated SDF delays.

The following was attempted:
1. OpenROAD PSM IR drop analysis was run and completed successfully (included
   in the OpenLane flow log). It provides supply current distribution but not
   a full dynamic power breakdown.
2. VCD generation was attempted via QuestaSim $dumpvars but the gate-level
   netlist (compute_core.nl.v) requires the sky130_fd_sc_hd liberty models to
   be mapped in the simulator. The liberty files are available in the OpenLane
   PDK installation but were not accessible from the local QuestaSim environment
   on Windows 11 without additional path configuration.

Dynamic Power Estimate (activity-based approximation):
  Assumptions:
    - Average switching activity: 0.20 (20% of nets toggle per cycle)
    - Supply voltage: 1.8 V
    - Clock frequency: 100 MHz
    - Total net capacitance (from SPEF): ~8.2 pF estimated from wire length
      (23,592 µm at sky130 ~0.35 fF/µm = 8.26 pF)

  Dynamic power = 0.5 × C × V² × f × activity
               = 0.5 × 8.26e-12 × 1.8² × 100e6 × 0.20
               = 0.5 × 8.26e-12 × 3.24 × 100e6 × 0.20
               ≈ 267 µW

  Total estimated power: 267 µW (dynamic) + 139 µW (leakage) ≈ 406 µW

  Energy per MAC operation (at 100 MHz, 1 MAC/cycle):
    406 µW / 100e6 ops/s = 4.06 pJ/MAC
